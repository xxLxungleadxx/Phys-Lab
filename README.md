## 概要

Phys-Lab は、高校生が物理の授業・復習・自学で使える教材をまとめるための Web サイトです。

Google Sites で運用していた教材置き場を GitHub Pages へ移行し、HTML ベースの教材や資料を少しずつ整備していきます。

## 対象

- 高校物理を学ぶ生徒
- 授業内容を自宅で見返したい生徒
- テスト前に要点や例題を確認したい生徒
- 物理の現象を図・グラフ・シミュレーションで確かめたい生徒

## サイトの役割

高校生がトップページを見たときに、「どこに何があるか」がわかることを重視します。

教材は、作成者側の分類ではなく、生徒が使う場面を意識して整理します。

- 授業で扱った資料をもう一度確認する
- 公式や考え方を読み直す
- 問題を解く前に条件や流れを整理する
- シミュレーションを動かして現象のイメージをつかむ
- 発展学習や調べものに使えるリンクを探す

## 掲載内容

### 授業用スライド

授業で扱った図、説明、演習の流れを見返すための資料です。欠席した回の確認や、ノート整理にも使えるようにします。

### PDF ノート

公式、考え方、例題の流れをまとめた読み物です。授業後の復習やテスト前の確認に使えることを想定します。

### 動かせる教材

HTML ベースのシミュレーションや確認ツールです。力、運動、波、電気などの内容を、グラフや操作を通して確かめられる教材を作ります。

### リンク集

授業内容を別の説明で確認したいときや、発展的に調べたいときに使える外部資料を整理します。

## 作成方針

- 高校生が迷わず使えるトップページにする
- 物理が苦手な生徒にも入りやすい表現を心がける
- 公式の暗記だけでなく、図・グラフ・現象とのつながりを重視する
- 授業で実際に使える教材を優先して作成する
- GitHub Pages で公開できる静的な HTML/CSS/JavaScript を基本とする

## ページ作成の基本方針

- 共通の見た目は `style.css` にまとめ、各 HTML で個別調整を増やしすぎない
- 各ページは `site-header`、`section`、`section-head`、`page-title`、`section-text` を基本構成にする
- PC 画面では `.page-title` が不要に改行されないよう、CSS 側で制御する
- スマートフォンやタブレットでは、横スクロールを前提にせず、自然に折り返して読める構成にする
- 新しい教材カテゴリを追加するときは、トップページと該当カテゴリページの両方からたどれるようにする

## シミュレーション作成の基本方針

- 静的な HTML/CSS/JavaScript だけで動作する形を基本とする
- canvas を使う場合は、CSS 上の表示サイズと `canvas.width` / `canvas.height` の内部解像度を分けて扱う
- GitHub Pages 上でも表示が崩れないよう、canvas の表示サイズは親要素幅を基準に明示的に制御する
- 物理量の表示範囲は、必要に応じてユーザーがスライダーで変更できるようにする
- 軸の範囲、現在値、代表値は、グラフだけでなく数値でも確認できるようにする
- 操作パネルは、初速度・角度・重力加速度・軸範囲など、授業中に比較したい量を優先して置く

## 表示確認の基本方針

- ローカル環境では `python3 -m http.server` などで配信して確認する
- GitHub Pages で公開した後、PC 画面とスマートフォン幅の両方で確認する
- 特に canvas、見出し、ナビゲーション、PDF リンクは、ローカルと GitHub Pages で表示差が出ないか確認する
- 変更後は `git diff --check` で不要な空白や基本的な差分エラーを確認する

## 開発方針

- Codex によるコーディングを主軸とする
- 必要に応じて NeoVim や手動編集で調整する
- README.md も、サイトの方向性に合わせて随時更新する
- 公開物が増えたら、トップページから探しやすい構成へ整理する
- 公開用と開発用で **branchを分ける**
    - main: 公開用
    - dev:開発用
